- props はあくまで親componentから子componentへ値を渡すためのものなので、ひとつのcomponentでだけ使う値の定義にはふさわしくないはず (使えるけども)
- ひとつの component の中でだけ利用する ローカルな propsという表現自体が本末転倒的な

# 例

こんな props は

```
export default {
  props: {
    count: {
      type: Number,
      default: 1
    },
  },
```

こんな風にdataに置き換える

```
export default {
  data () {
    return {
      count: 1
    }
  },
```
